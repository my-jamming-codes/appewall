**This article demonstrates how to use google auto library to generate methods at compile time using
java annotation processors**

## Google Auto Library

- Add dependency

```xml
<dependency>
    <groupId>com.google.auto.service</groupId>
    <artifactId>auto-service</artifactId>
    <version>1.0-rc7</version>
    <scope>provided</scope>
</dependency>
```

## Write an annotation processor

- Loggable annotation processor

```java
---
---
package com.example;

import com.google.auto.service.AutoService;
import javax.annotation.processing.AbstractProcessor;
import javax.annotation.processing.RoundEnvironment;

@AutoService(Processor.class)
public class LoggableProcessor extends AbstractProcessor {
    @Override
    public boolean process(Set<? extends TypeElement> annotations, RoundEnvironment roundEnv) {
        // Implement your annotation processing logic here
        return false;
    }
}

```

## Include the annotation processor in the build

- Add the annotation processor to the `META-INF/services/javax.annotation.processing.Processor` file

```java

com.example.LoggableProcessor
```

## Conclusion

- Annotation processors are a powerful tool for generating code at compile time. The Google Auto library simplifies the process of writing annotation processors by providing a convenient API for registering processors and generating code.
